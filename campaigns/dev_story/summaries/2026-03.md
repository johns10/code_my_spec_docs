# March 2026: The Harness Rebuilds Its Own Foundation

March was the month CodeMySpec was forced to eat its own cooking. While MetricFlow churned through its QA crunch, every gap in the harness surfaced as a real problem in a real product, and 188 commits worth of fixes, replacements, and rewrites poured back into the framework that had built it. The month began with iteration on the existing model and ended with that model thrown out and rebuilt from a different angle entirely.

The opening week was about giving agents a way to talk back. An Issues context landed -- schema, repository, projector, and an MCP server with a create_issue tool so subagents could file their own reports during runs. A triage and fix-issues pipeline was wired up alongside it: severity defaults dropped to :info so nothing got silently dismissed, and the QA story lifecycle was overhauled to gate on result_complete.md. The permission request system arrived in the same stretch -- a Phoenix channel, a notification service worker, a LiveView for approving remote Claude Code tool calls -- so the harness could run claude on a remote box and ask the user before a destructive shell command. By March 4 the BDD specs had been converted from LiveViewTest to Wallaby browser testing, which is what the harness now demanded of every project it generated.

Mid-month brought the incremental sync pipeline. The full-tree resyncs that had powered earlier validation were replaced with mtime-filtered sync feeding a three-phase validation, and a DirtyTracker on components made repeated runs cheap. Problems gained component_id so analyzer output could be routed back to the component that owned the file. ExCliVcr cassettes spread across the validation and project_setup test suites; the cassette files themselves grew, shrank, and grew again as the underlying pipelines were rewritten. QA journey tasks (plan, execute, wallaby) shipped with their own checkers, and a framework Credo check landed -- with a manifest-based auto-discovery mechanism so projects could add their own checks without touching the harness.

The week of March 21 was the inflection. A new CodeMySpecLocalWeb Phoenix endpoint appeared next to the existing web app -- a LiveView-and-API split running on its own port, served from the user's machine, with a ProjectScope plug, a sync page, requirement views rendered through MDEx instead of Earmark, and a requirements graph visualization driven by a depends_on edge type and a GraphProjector. Hermes MCP was swapped out for Anubis. Sessions moved from in-memory stacks to a DB-backed task lifecycle with validation_type and execution_type fields, an evaluate endpoint, and a PreToolUse hook that enriched tool input with session context. Requirements stopped being a flat list of artifact-existence checks and became a task graph the agent could navigate.

Then late March took it further. Across two days the legacy local_server namespace was deleted -- 2,800 lines of REST controllers and plugs replaced by MCP tools (issues, stories, tasks, bootstrap, requirements) running through a single MCP plug, with controllers and hooks migrated to local_web. The agent task interface was refactored to command(scope, task) instead of command(scope, session), touching every one of the forty-odd agent task modules. ProjectSetup was extracted into per-concern step modules, and the Init task got its own pre-project-ID bootstrap chain. A dev_cli environment and a Burrito-built prod_cli release made the whole thing shippable as a single binary against SQLite, and the Claude Code plugin -- agents, skills, hooks, knowledge files -- moved into the repo itself.

The last weekend was the sharpest turn. A new Files schema and a FileSync pipeline landed, validating files inline as the watcher saw them. Then a RequirementCalculator and a RequirementGraph appeared -- computing satisfaction directly from Files plus Problems, with a Preloader to kill N+1 queries and a 20x speedup on graph computation. Once that worked, the old machinery came down: 3,963 lines of legacy sync, validation pipeline, and project_sync deleted in one commit, then 3,029 lines of requirement checkers -- 27 separate checker modules -- deleted in the next. The framework that had spent the previous year shaped around imperative checkers running over a flat requirements list now derived everything from a computed graph over a Files table the watcher kept current.

It was the same pattern MetricFlow's QA cycle had been forcing all month: when the abstraction stops fitting, throw it out, not patch it. The harness ended March smaller in line count than it started, doing more, with the product it had built running through it daily.
