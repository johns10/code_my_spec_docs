# March 2026: The QA Pipeline Matures and Issues Earn Their Own Home

The first week of March arrived with the intensity of a system reaching critical mass. Across 19 commits in just six days, CodeMySpec underwent a sweeping overhaul of its quality assurance infrastructure, introduced a full-blown Issues context, and wired up a permission system that let remote AI agents ask humans for approval before acting.

The month opened on March 1st with the kind of commit that reshapes a product's trajectory. A new issue triage and fix pipeline landed alongside a complete overhaul of the QA story lifecycle -- nearly 1,750 lines added across 25 files. `TriageIssues` and `FixIssues` agent tasks arrived as first-class session types, and the requirements system gained `IncomingIssuesChecker` and `AcceptedIssuesChecker` to gate issue flow through formal requirement definitions. The QA story agent was rewritten to manage its own directory structure, and a login tool appeared in the priv/tools directory, hinting at the system's growing need to authenticate against real applications during testing.

March 2nd was a single, surgical commit: OAuth configuration was simplified by replacing a bespoke `oauth_base_url` config with `Endpoint.url()`, a quiet nod to the Cloudflare tunnel deployment model the project had adopted. Small commits like these revealed a developer cleaning house between larger pushes.

March 3rd exploded. A mass rename migrated every test support module into the `CodeMySpecTest` namespace -- 126 files touched in a single commit, a refactoring chore that most teams defer indefinitely. But the real payload arrived in the commits that bookended it: a `ProjectBootstrap` agent task (585 lines), a `ResearchTopics` orchestrator, new knowledge documents for Wallaby browser testing, Req HTTP cassettes, and dotenvy configuration. The `ProjectCoordinator` gained dispatch logic for QA workflows, and the Stories context was simplified. Infrastructure kept pace -- the application module was reorganized, dependencies updated, and a full `structure.sql` was committed for the first time.

March 4th belonged to QA tooling. Vibium replaced the previous web testing tool in the knowledge docs. BDD specs were converted from LiveViewTest to Wallaby browser testing, a shift that acknowledged the limits of in-process test rendering for real UI validation. The `develop_*` agent tasks were dramatically simplified -- over 4,200 lines deleted -- as the system consolidated around a shared implementation pattern and introduced `FixBddSpecs` as a dedicated recovery task. A Credo path-doubling bug was squashed, and test race conditions were tamed.

March 5th brought the most architecturally novel feature of the week: a permission request system. A `PermissionController`, `PermissionSocket`, LiveView interface, and Phoenix channel were wired together so that remote Claude Code sessions could pause, push a notification, and wait for a human to approve or deny a tool invocation. The service worker was updated to handle push notifications. It was the kind of feature that only makes sense when AI agents are running autonomously against real codebases -- a trust boundary made tangible. The same day saw `FrameworkSync` introduced and unique constraints added to requirements.

The week closed on March 6th with the Issues context crystallizing into its own bounded context: a dedicated `Issues` module with schema, repository, and projector -- 2,552 lines added. The QA and triage agents were refactored to work through this new context boundary, and the devops knowledge base was condensed. Issues had graduated from document parser artifacts into proper domain entities with their own lifecycle.

The pace was relentless -- three or four substantial commits per day, each building on the last. What emerged was a system no longer just generating and validating code, but one actively managing the feedback loop: finding issues, triaging them, requesting human permission when needed, and tracking resolution through a formal domain model. CodeMySpec was becoming its own quality department.
