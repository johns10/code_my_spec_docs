# July 2025: Genesis

July 2025 was the month CodeMySpec came alive. Across 29 commits, a solo developer laid down every foundational layer of the application -- from the very first `mix phx.new` to a working MCP server with OAuth-protected endpoints. The pace was relentless, with the heaviest activity concentrated in two explosive bursts: mid-month and the final week.

It began on July 1st with an initial commit and a demo generator Mix task, a sign that even from day one the project was thinking about showcasing its own capabilities. Within days, the early architectural scaffolding for an agent system appeared: an `Agent` schema, a behaviour module, a type registry, and the beginnings of a Claude Code CLI adapter. These first commits revealed the project's ambition -- not just another web app, but a platform designed to orchestrate AI coding agents.

By mid-July, development shifted into full-stack construction mode. The project upgraded to Phoenix 1.8, pulling in DaisyUI and modernizing the component layer in a single sweeping commit that touched 18 files. Authentication landed the same day via `mix phx.gen.auth`, instantly delivering user registration, login, sessions, and a complete test suite. What followed was a rapid vertical build-out: Accounts with member roles and an authorization module, user preferences with a LiveView settings form, an invitation system complete with email notifications and acceptance flows, and a Project context with CRUD, breadcrumbs, and a picker component. Each feature arrived fully formed -- context module, repository, schema, migration, LiveView UI, and tests -- all in a single commit. The Scope struct evolved in lockstep, growing to carry user, account, and project context through the system.

The final week of July belonged to Stories and MCP. On July 22nd, a massive 1,340-line commit introduced the Stories context: schema, repository, LiveView index/show/form, and comprehensive tests. The very same day, an OAuth2 implementation landed -- grants, tokens, applications, an authorization controller with a consent screen. Then the MCP server build began in earnest. Over four rapid-fire commits on July 25th and 26th, the Stories MCP server grew from a single `create_story` tool to a full suite: create, update, delete, list, get, bulk create, plus interview and review prompts. Resources were tried and then deliberately converted to tools in a refactoring commit that showed the developer learning the MCP protocol's strengths in real time. A mapper layer emerged to translate between internal schemas and MCP representations.

The month closed with two more significant additions. A markdown import system arrived on July 27th, allowing stories to be parsed from structured markdown files and imported through a LiveView interface. And on July 30th, a Components context materialized -- components with dependencies, tracked in the database -- adding the first hints of the architectural modeling that would become central to the product's identity.

Twenty-nine commits. Over 18,000 lines of production code. A full authentication system, multi-tenant accounts, invitations, projects, stories, an MCP server, OAuth, and the seeds of an agent orchestration layer. July was not a month of planning. It was a month of building at speed, with each commit arriving test-covered and structurally complete. The foundation was set.
