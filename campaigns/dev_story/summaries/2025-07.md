# July 2025: Genesis

July was the month CodeMySpec went from an empty Phoenix project to a working multi-tenant application with stories, OAuth, and the first MCP endpoints. The first commit landed on July 1 -- a vanilla `mix phx.new` with a `generate_demo` mix task already sketched in. By the end of the month, twenty-eight commits later, the project had agents, accounts, projects, stories, components, and a story interview tool exposed over MCP.

The earliest work was reconnaissance. A demo generator mix task arrived on day one, then sat idle for two days while attention shifted to agents. The first agent component file appeared on July 5 -- an `Agent` struct, an `AgentBehaviour`, and a registry that resolved agent types. This was the seed of everything: the project would treat coding agents as pluggable implementations behind a common interface, and the rest of the system would grow around the question of what to feed them. Four days later the Claude Code adapter arrived, with a CLI adapter behind a behaviour and a `ClaudeCodeRecorder` test helper that could replay recorded CLI sessions. The first piece of test infrastructure aimed at agents specifically.

Mid-July pivoted hard to the user-facing platform. Phoenix 1.8 came in on July 14, immediately followed by `mix phx.gen.auth` and a daisyUI overhaul. The next morning the Accounts context landed in a single 1,935-line commit -- accounts, members, repositories, an Authorization module, and over a thousand lines of tests. User preferences shipped that afternoon. On July 16 the Invitations context arrived at similar scale: 1,552 lines covering schema, repository, notifier, and acceptance flow. The pattern was set early. Contexts dropped in fully formed with their tests, repository module, and LiveView shell, and the `Scope` struct quietly grew to carry user, account, and project through the system.

The last ten days of July were where the project's actual thesis began to surface. Projects landed on July 18 with picker, breadcrumb, and scope plumbing. OAuth2 followed the same day -- access grants, access tokens, an authorize controller -- because the MCP server would need it. Stories arrived on July 22 with versioning support. Then on July 25 the first MCP endpoint shipped: a stories MCP server with a `create_story` tool, formatters, validators, and the first integration test against an MCP client. By July 26 every story operation had been "converted to tools" -- the prompt and resource scaffolding was abandoned in favor of tool calls, an early sign that the developer was learning the protocol's grain in real time. Markdown import for stories shipped on July 27, components and dependencies on July 30, and the story-to-component association on July 31. The graph was starting to form.
