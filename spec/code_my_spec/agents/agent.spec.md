# CodeMySpec.Agents.Agent

Embedded Ecto schema representing an AI agent configuration. Agents combine a name, implementation type, agent type definition, and configuration to define how AI assistants interact with the system.

## Fields

| Field          | Type      | Required | Description                                | Constraints                          |
| -------------- | --------- | -------- | ------------------------------------------ | ------------------------------------ |
| id             | binary_id | Yes      | Unique identifier for the agent            | Auto-generated by embedded_schema    |
| name           | string    | Yes      | Display name for the agent                 |                                      |
| agent_type     | AgentType | Yes      | Embedded agent type configuration          | on_replace: :update                  |
| implementation | enum      | Yes      | Agent implementation backend               | Values: [:claude_code]               |
| config         | map       | No       | Additional agent configuration             | Default: %{}                         |

## Functions

### changeset/2

Validates and casts agent attributes for create/update operations.

```elixir
@spec changeset(t(), map()) :: Ecto.Changeset.t()
```

**Process**:
1. Cast name, implementation, and config fields from input attributes
2. Extract agent_type from attributes (defaulting to empty AgentType struct)
3. Put the embedded agent_type association
4. Validate that name, agent_type, and implementation are required

**Test Assertions**:
- returns valid changeset with valid name, agent_type, and implementation
- returns invalid changeset when name is missing
- returns invalid changeset when implementation is missing
- returns invalid changeset when agent_type is missing
- accepts :claude_code as valid implementation value
- returns invalid changeset for unknown implementation values
- uses empty map as default config when not provided
- preserves existing config when updating
