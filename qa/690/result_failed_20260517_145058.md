## Issues

### MarkdownBrowserLive crashes with FunctionClauseError on all knowledge routes

#### Severity
HIGH

#### Scope
APP

#### Description
`lib/code_my_spec_local_web/live/markdown_browser_live.ex` line 134 has:
```elixir
defdelegate resolve_root(root, project_name), to: Knowledge
```
This delegates to `Knowledge.resolve_root(root, project_name)`. The LiveView calls it at
line 35 as `resolve_root(config.root, project_name)`, which passes `({:priv, "knowledge"}, "code-my-spec")` to `Knowledge.resolve_root/2`. However, that function expects `(scope :: Scope.t(), root_spec)` — the arguments are in the wrong order and the scope is missing entirely.

Every knowledge route in the LiveView crashes:
- `GET /projects/:project_name/knowledge`
- `GET /projects/:project_name/knowledge/*path`
- `GET /projects/:project_name/project-knowledge`
- `GET /projects/:project_name/project-knowledge/*path`

The API controller (`MarkdownBrowserController`) uses `Scope.for_local_project(project)` correctly and is not affected. The MCP tools are not affected.

Fix: replace the `defdelegate` with a local helper:
```elixir
defp resolve_root(root_spec, project_name) do
  with {:ok, project} <- Projects.get_project_by_name(project_name),
       {:ok, scope} <- Scope.for_local_project(project) do
    Knowledge.resolve_root(scope, root_spec)
  end
end
```
